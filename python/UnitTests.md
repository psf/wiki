# Python Testing Frameworks {#Python_Testing_Frameworks}

- [Unittest](https://docs.python.org/3/library/unittest.html){.https} \-- full-featured unit testing system, inspired by Java\'s JUnit. Part of the Python Standard Library (as an external project it was called [PyUnit](PyUnit)).

- [DocTest](https://docs.python.org/3/library/doctest.html){.https} \-- lightweight testframework, allows you to embed tests directly in the docstrings, making it easy to combine code examples and test cases in one place. Part of the Python Standard Library.

- [PyTest](https://docs.pytest.org){.https} \-- alternative to [PyUnit](PyUnit) and with a simpler syntax.

- [nose2](https://docs.nose2.io/en/latest/){.https} \-- a discovery-based framwork that extends unittest (successor to `nose`{.backtick}).

- [Testify](https://github.com/Yelp/Testify/wiki){.https} \-- Pythonic testing framework compatible with unittest tests.

- [Robot](https://robotframework.org/){.https} \-- Framework for test automation and robotic process automation (not just for Python).

- [Behave](https://pypi.org/project/lettuce/){.https} \-- Framework for Behavior-Driven Development.

- [Hypothesis](https://hypothesis.works/){.https} \-- Powerful property-based testing.

These are useful for [TestDrivenDevelopment](TestDrivenDevelopment)

# Version Control {#Version_Control}

The following applications related to version/revision control are written in Python:

## Version Control Systems {#Version_Control_Systems}

- [Bazaar](http://bazaar-vcs.org/){.http} (formerly Bazaar-NG)

- [Codeville](http://codeville.org/){.http}

- [Mercurial](http://www.selenic.com/mercurial/){.http}

## Abstraction Layers and Tools {#Abstraction_Layers_and_Tools}

Tools with unified interface to several version control systems.

- [Hg-Git](http://hg-git.github.com/){.http} - access Git repositories from Mercurial

- [pysync](http://code.google.com/p/pysync/){.http} - abstraction layer, looks inactive, but a lot of commits

- [pyvcs (Alex Gaynor and co.)](https://github.com/alex/pyvcs){.https} - pure python abstraction layer of multiple VCS, very lightweight

- [vcs](http://pypi.python.org/pypi/vcs/){.http} - another abstraction layer, seems to be MIT licensed

## Migration Tools {#Migration_Tools}

- [cvs2svn](http://cvs2svn.tigris.org/){.http} - the CVS to Subversion repository converter

- [Tailor](http://progetti.arstecnica.it/tailor){.http} - a tool to migrate changesets between ArX, Bazaar, Bazaar-NG, CVS, Codeville, Darcs, Git, Mercurial, Monotone, Subversion and Tla repositories

## Repository Browsers {#Repository_Browsers}

- [Trac](http://trac.edgewall.org/){.http} supports repository browsing

- [ViewVC](http://www.viewvc.org/){.http} (formerly ViewCVS) - a repository browser for CVS and Subversion

  [pdb](https://docs.python.org/library/pdb.html){.https}                                                           All                       The standard library debugger, part of all Python installations.
  [pudb](https://pypi.python.org/pypi/pudb){.https}                                                                 Unix,Mac OS X             A visual, console-based, full-screen debugger, designed as a more comfortable drop-in replacement for pdb. (also supports IPython)
  [seapie](https://github.com/hirsimaki-markus/seapie){.https}                                                      All (CPython)             A REPL-first debugger where breakpoint drops into a \>\>\> prompt at the current execution state. Debugging is done directly in the REPL, with !commands for stepping and frame control. Changes to variables and function definitions persist across execution.
  [PdbRcIdea](PdbRcIdea)                                                                                     All                       A .pdbrc for Python\'s standard debugger, pdb, which allows you to run arbitrary Python commands on pdb startup.
  gdb                                                                                                               \*nix                     See [DebuggingWithGdb](DebuggingWithGdb)
  [trepan2](https://pypi.python.org/pypi/trepan2){.https}, [trepan3k](https://pypi.org/project/trepan3k/){.https}   Unix, Windows, Mac OS X   A rewrite of pdb/pydb with closer compliance to gdb . This is the only debugger that I (rocky) am aware of that uses decompilation technology (also written by me), so that you can debug CPython bytcode files where no source code is available. Decompilation is also used to provide sensible debugging inside exec strings, and more accurate position information when stopped or in showing a stack trace. In addition to the features of pdb/pydb, the debugger supports syntax coloring (via pygments), has extensive on-line help (in rendered [reStructuredText](reStructuredText)), [readthedocs documentation](https://python3-trepan.readthedocs.io/en/latest/){.https}, command completion, a smarter eval, debugger macros written in Python, and more.
  [pdb++](https://github.com/antocuni/pdb){.https}                                                                  Unix, Windows, Mac OS X   An extension of the pdb module of the standard library. It is meant to be fully compatible with its predecessor, yet it introduces a number of new features to make your debugging experience as nice as possible.
  [Python-pydebug](https://github.com/benmezger/pydebug){.https}                                                    Unix, Windows, Mac OS X   A set of debugging decorators which respects Django\'s settings in case the package is withing a Django project. It allows a user to PDB into a function, do a Line profiler, inspect an object and Disasemble the function. Archived/abandoned by author in 2023
  [winpdb-reborn](https://pypi.org/project/winpdb-reborn/){.https}                                                  Windows, Linux            Fork of winpdb after it went unmaintained. Project released a v2.0 which supports Python 3, but has not released anything since 2020.
  [ipdb](https://github.com/gotcha/ipdb){.https}                                                                    All                       Improved version of pdb that is part of IPython but also can be used separately
  [madbg](https://github.com/kmaork/madbg){.https}                                                                  linux, python\>=3.7       fully-featured remote debugger: attaching to running programs, especially usefull to debug programms with TUI

### IDEs with Debug Capabilities {#IDEs_with_Debug_Capabilities}

Since debugging is one of the the functions that usually helps make up the \"Integrated\" in \"Integrated Development Environment\", expect that most IDEs will have debugging capability even if not listed explicitly here (see [IntegratedDevelopmentEnvironments](IntegratedDevelopmentEnvironments)).

::: {}
  -------------------------------------------------------------------------------------------------------------- -------------------------------- ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
  [Thonny](https://thonny.org){.https}                                                                           Mac OS X, Linux, Windows         For teaching/learning programming. Focused on program runtime visualization. Provides stepping both in statements and expressions, no-hassle variables view, separate mode for explaining references etc.
  [PyDev](https://pydev.org){.https}                                                                             Mac OS X, Linux, Windows         Allows debugging multiple threads in Jython and Python (It is featured as a \'Python IDE\' plugin for Eclipse).
  [Wing IDE](https://wingware.com/){.https}                                                                      Mac OS X, Linux, Windows         An IDE that can debug multiple threads and multiple processes, including code launched from the IDE or code launched externally, running under CPython and Stackless Python. The GUI includes a Debug Probe, which is a Python shell running in the context of the paused debug process. The IDE\'s debugger also features value watching (by symbolic path, object reference, or a combination), conditional breakpoints, move program counter, debugging of tests running in the integrated unit testing tool, special support for Django, and How-Tos for debugging code running under Flask, web3py, Django, Google App Engine, wxPython, [PyQt](PyQt), Tkinter, Blender, Maya, NUKE, and many other packages.
  [Komodo IDE](https://activestate.com/komodo/features/){.https}                                                 Mac OS X, Linux, Windows         [ActiveState](ActiveState) have retired Komodo, the code is open sourced at [https://github.com/ActiveState/OpenKomodoIDE](https://github.com/ActiveState/OpenKomodoIDE){.https}
  [PyCharm IDE](https://www.jetbrains.com/pycharm/features/debugger.html){.https}                                Mac OS X, Linux, Windows         PyCharm\'s integrated debugger works for Python and Jython, supports debugging of multiple threads, remote debugging, allows debugging Django, Google App Engine applications and unit tests. The debugger features various breakpoints, stepping modes, frames view, watches, evaluate expression tool and a debug console. Conditional and Exception breakpoint types are available for more precise control. Debug console allows executing any Python statements in the context of the process being debugged while stopped at a breakpoint.
  [PyScripter](https://github.com/pyscripter/pyscripter){.https}                                                 Windows                          
  Pyshield                                                                                                       Linux, Windows                   An IDE tool used to edit, debug Python scripts, publish encrypted scripts, build a standalone executable file, and make installation in various forms(.msi, .tar.gz, .rpm, .zip, .tar.bz2). It includes an editor simulating Emacs python-mode, a GUI debugger simulating GDB, a project view used to manage scripts, modules, extensions, packages and platform specific data files. This seems to have disappeared, previous link was to a DashingSoft project called PyShield which is something different. Will remove if nobody finds a link.
  Python Tools for Visual Studio                                                                                 Windows                          Supports Python (any implementation with sufficient sys.settrace capabilities) and [IronPython](IronPython) .NET debugging. NOTE: Since VS 2017, this is integrated in VS and called Python Support in Visual Studio.
  [Visual Studio Code Debugger](https://marketplace.visualstudio.com/items?itemName=ms-python.debugpy){.https}   Mac OS X, Linux, Windows         Visual Studio Code is a source code editor developed by Microsoft for Windows, Linux and MacOS. It includes support for debugging, embedded Git control, syntax highlighting, intelligent code completion, snippets, and code refactoring. VS Code is free to use, although the official version is under a proprietary license. [VSCodium](https://vscodium.com){.https} is the unencumbered version, but it cannot use extensions from the Marketplace.
  [Mu](https://codewith.mu/){.https}                                                                             Mac OS X, Linux, Windows         A simple Python editor for beginner programmers, providing a simple interface depending on the project type. It includes visual debugging as first citizen.
  [Jupyter](https://jupyter.org/){.https}                                                                        Web (Mac OS X, Linux, Windows)   Web UI for Notebooks in Python and other programming language, has a [visual debugger](https://blog.jupyter.org/a-visual-debugger-for-jupyter-914e61716559){.https} intregrated.

### Special-purpose tools {#Special-purpose_tools}

::: {}
  -------------------------------------------------------------------------------------------- ------------------------- ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
  [manhole](https://github.com/ionelmc/python-manhole/){.https}                                Mac OS X, Linux           Remote process inspector (using an active component, using a thread or a plain simple signal handler).
  [pdb-clone](https://pypi.python.org/pypi/pdb-clone){.https}                                                            Remote process inspector. Uses GDB to inject code.
  [PyConquer](./PyConquer.html){.nonexistent}                                                  Any                       Trace hook logger which outputs each thread in its own \"swimlane\" to make multithreaded analysis easier. Can also time calls (naively) and watch variables. The former site is a now a malware trap. If the tool still exists, please update with a valid link.
  [pydbattach](https://github.com/albertz/pydbattach){.https}                                                            Remote process inspector. Uses GDB to inject code.
  [pyrasite](https://pypi.python.org/pypi/pyrasite){.https}                                    Mac OS X, Linux           Library and a set of tools for injecting code into running Python programs to monitor, analyze, introspect, and alter running Python programs easily. Uses GDB to inject code.
  [pyringe](https://github.com/google/pyringe){.https}                                                                   Remote process inspector. Uses GDB to inject code.
  [pystuck](https://github.com/alonho/pystuck){.https}                                                                   Remote process inspector. Uses GDB to inject code. Uses [rpyc](https://github.com/tomerfiliba/rpyc){.https} for communication.
  [wsgi-shell](https://github.com/GrahamDumpleton/wsgi-shell){.https}                                                    Remote process inspector (using an active component, a thread). This was renamed to `ispyd`{.backtick} and marked abandoned by the author in 2020
  [hunter](https://github.com/ionelmc/python-hunter){.https}                                                             A flexible code tracing toolkit. Can print out code and variables, and filter the events.
  [icecream](https://github.com/gruns/icecream/){.https}                                                                 Sweet and creamy print debugging \-- inspect variables, expressions, and code execution with a single, simple function call.
  [birdseye](https://birdseye.readthedocs.io){.https}                                                                    Graphical Python debugger which lets you easily view the values of all evaluated expressions
  [peepshow](https://github.com/gergelyk/peepshow){.https}                                                               Provides two utilities: **show**: a lightweight function that prints name and value of your variable(s) to the console, and **peep**: featured, interactive interface for data inspection.
  [PySnooper](https://github.com/cool-RR/PySnooper/){.https}                                                             Never use print for debugging again. PySnooper is a poor man\'s debugger. If you can\'t use a real debugger for some reason or other, and you\'re resorting to adding print statements to your code, this is for you. Add just one decorator line to your functions, and you get a complete log of all the lines that ran in the function and all the variables that were changed.
  [snoop](https://github.com/alexmojaki/snoop){.https}                                                                   An improved version of PySnooper.
  [VizTracer](https://github.com/gaogaotiantian/viztracer){.https}                             Unix, Windows, Mac OS X   Trace and visualize your function execution on a graphical UI, with filters and arbitrary data logging support. Featured with a virtual debugger like pdb to debug your log of execution
  [Python Step-by-Step Online Debugger](https://first.institute/edu/python-sandbox/){.https}   Web-browser               Records step-by-step execution of a Python program. Shows values of local variables in a table with ability to select every step. Works in a web-browser. Supports Python 3.12.

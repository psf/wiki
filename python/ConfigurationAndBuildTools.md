## 1.1. Dead, Unmaintained or Discontinued Projects {#Dead.2C_Unmaintained_or_Discontinued_Projects}

- [AAP](http://www.a-a-p.org/){.http} - Latest version 1.093 released on 2013-09-15. Does not appear to support Python 3.

- [Bake](http://projects.bertram-scharpf.de/bake/index.html){.http} - a Make replacement. Website is down, [archived here](http://web.archive.org/web/20071007134639/http://projects.bertram-scharpf.de/bake/index.html){.http}. No release in the last 10 years.

- [BuildIt](http://agendaless.com/Members/chrism/software/buildit/README.txt){.http} (by chrism) - Build tool for creating applications. Website down, [web.archive.org](http://web.archive.org/web/20120114061845/http://agendaless.com/Members/chrism/software/buildit/README.txt){.http}. No release in the last 10 years.

- [BuildIt](https://bitbucket.org/mixmastamyk/buildit){.https} (by mixmastamyk) - [BuildIt](./BuildIt.html){.nonexistent} is a simple build tool and make replacement, meaning it builds files based on other files, given your instructions. Website down, [web.archive.org](http://web.archive.org/web/20161008052108/https://bitbucket.org/mixmastamyk/buildit){.http}. [pypi.org](https://pypi.org/project/buildit/){.https}. Last release in 2016.

- [Confix](http://confix.sourceforge.net/){.http} - generates input files for [automake](http://www.gnu.org/software/automake/){.http}. [sourceforge.net](https://sourceforge.net/projects/confix/){.https}. Website down, [web.archive.org](http://web.archive.org/web/20080429230516/http://confix.sourceforge.net:80/){.http}. Last release was 2.3.9 on 2013-01-29, does not support python 3. [sourceforge.net](https://sourceforge.net/projects/confix){.https}

- [Memoize](http://www.cs.berkeley.edu/~billm/memoize.html){.http} - Designed as a replacement for Make - *site unavailable, see the [Internet Archive version](http://web.archive.org/web/20090528032849/http://www.cs.berkeley.edu/~billm/memoize.html){.http}*

- [PyAnt](https://sourceforge.net/projects/pyant/){.https} - *development discontinued*, last release 0.32 was on 2003-04-12.

- [PyBuilder](http://pybuilder.sf.net){.http} (David Mack)- A Gui interface to compile scripts using py2exe. Last release 0.3 was on 2007-02-28.

- [PyMake](http://www.plope.com/software/pymake){.http} ([web.archive.org](http://web.archive.org/web/20170201233400/http://plope.com/software/pymake/){.http}) - Designed as a replacement for Make. *Development replaced by [BuildIt](./BuildIt.html){.nonexistent}*.

- [PyMake2](https://bitbucket.org/saudalwasly/pymake2/){.https}, site down and last release 0.5.33 on 2017-10-28 [pypi.org](https://pypi.org/project/pymake2/){.https}

- [Rebuild](http://staff.bbhc.org/christophern/exos/rebuild/index.html){.http} - python module that replaces make - *site unavailable, see the [web.archive.org](http://web.archive.org/web/20041214162327/http://staff.bbhc.org:80/christophern/exos/rebuild/index.html){.http}*

- [Vellum](http://www.zedshaw.com/projects/vellum/){.http} ([web.archive.org](http://web.archive.org/web/20090201180349/http://www.zedshaw.com:80/projects/vellum/){.http}) - Simple build tool similar to Make. *Dead project. [https://github.com/zedshaw/vellum](https://github.com/zedshaw/vellum){.https}*. Last release 0.17 on 2009-06-30.

- [Vertoo](http://vertoo.sourceforge.net/){.http} - **ver**sion management **too**l (automates version number maintenance across several files). Last release 0.8 on 2003-07-18. [sf.net](https://sourceforge.net/projects/vertoo/){.https}

# 2. Build Testing {#Build_Testing}

- [Bitten](http://bitten.edgewall.org/){.http} - a continuous integration plugin for Trac

- [BuildBot](http://buildbot.sourceforge.net/){.http} The [TwistedMatrix](TwistedMatrix) project has a [BuildBot](BuildBot) installed which runs the unit test suite on a variety of platforms. The [BuildBot](BuildBot) uses the Twisted framework for networking, RPC, web status delivery, persistence, and other platform needs.

- [Apache Gump](http://gump.apache.org/){.http} - Apache\'s continuous integration tool. \"It is written in python and fully supports Apache Ant, Apache Maven and other build tools.\"

# 4. Editorial Notes {#Editorial_Notes}

The above lists should be arranged in ascending alphabetical order - please respect this when adding new entries. If, as the developer of a listed application, you disagree with the classification of the work, please move it into the appropriate category or create a new category, respecting the ascending alphabetical order of the categories. When specifying release dates please use the format YYYY-MM-DD.

## 1.1. Dead, Unmaintained or Discontinued Projects 

- [AAP](http://www.a-a-p.org/) - Latest version 1.093 released on 2013-09-15. Does not appear to support Python 3.

- [Bake](http://projects.bertram-scharpf.de/bake/index.html) - a Make replacement. Website is down, [archived here](http://web.archive.org/web/20071007134639/http://projects.bertram-scharpf.de/bake/index.html). No release in the last 10 years.

- [BuildIt](http://agendaless.com/Members/chrism/software/buildit/README.txt) (by chrism) - Build tool for creating applications. Website down, [web.archive.org](http://web.archive.org/web/20120114061845/http://agendaless.com/Members/chrism/software/buildit/README.txt). No release in the last 10 years.

- [BuildIt](https://bitbucket.org/mixmastamyk/buildit) (by mixmastamyk) - [BuildIt](./BuildIt.html) is a simple build tool and make replacement, meaning it builds files based on other files, given your instructions. Website down, [web.archive.org](http://web.archive.org/web/20161008052108/https://bitbucket.org/mixmastamyk/buildit). [pypi.org](https://pypi.org/project/buildit/). Last release in 2016.

- [Confix](http://confix.sourceforge.net/) - generates input files for [automake](http://www.gnu.org/software/automake/). [sourceforge.net](https://sourceforge.net/projects/confix/). Website down, [web.archive.org](http://web.archive.org/web/20080429230516/http://confix.sourceforge.net:80/). Last release was 2.3.9 on 2013-01-29, does not support python 3. [sourceforge.net](https://sourceforge.net/projects/confix)

- [Memoize](http://www.cs.berkeley.edu/~billm/memoize.html) - Designed as a replacement for Make - *site unavailable, see the [Internet Archive version](http://web.archive.org/web/20090528032849/http://www.cs.berkeley.edu/~billm/memoize.html)*

- [PyAnt](https://sourceforge.net/projects/pyant/) - *development discontinued*, last release 0.32 was on 2003-04-12.

- [PyBuilder](http://pybuilder.sf.net) (David Mack)- A Gui interface to compile scripts using py2exe. Last release 0.3 was on 2007-02-28.

- [PyMake](http://www.plope.com/software/pymake) ([web.archive.org](http://web.archive.org/web/20170201233400/http://plope.com/software/pymake/)) - Designed as a replacement for Make. *Development replaced by [BuildIt](./BuildIt.html)*.

- [PyMake2](https://bitbucket.org/saudalwasly/pymake2/), site down and last release 0.5.33 on 2017-10-28 [pypi.org](https://pypi.org/project/pymake2/)

- [Rebuild](http://staff.bbhc.org/christophern/exos/rebuild/index.html) - python module that replaces make - *site unavailable, see the [web.archive.org](http://web.archive.org/web/20041214162327/http://staff.bbhc.org:80/christophern/exos/rebuild/index.html)*

- [Vellum](http://www.zedshaw.com/projects/vellum/) ([web.archive.org](http://web.archive.org/web/20090201180349/http://www.zedshaw.com:80/projects/vellum/)) - Simple build tool similar to Make. *Dead project. [https://github.com/zedshaw/vellum](https://github.com/zedshaw/vellum)*. Last release 0.17 on 2009-06-30.

- [Vertoo](http://vertoo.sourceforge.net/) - **ver**sion management **too**l (automates version number maintenance across several files). Last release 0.8 on 2003-07-18. [sf.net](https://sourceforge.net/projects/vertoo/)

# 2. Build Testing 

- [Bitten](http://bitten.edgewall.org/) - a continuous integration plugin for Trac

- [BuildBot](http://buildbot.sourceforge.net/) The [TwistedMatrix](TwistedMatrix) project has a [BuildBot](BuildBot) installed which runs the unit test suite on a variety of platforms. The [BuildBot](BuildBot) uses the Twisted framework for networking, RPC, web status delivery, persistence, and other platform needs.

- [Apache Gump](http://gump.apache.org/) - Apache\'s continuous integration tool. \"It is written in python and fully supports Apache Ant, Apache Maven and other build tools.\"

# 4. Editorial Notes 

The above lists should be arranged in ascending alphabetical order - please respect this when adding new entries. If, as the developer of a listed application, you disagree with the classification of the work, please move it into the appropriate category or create a new category, respecting the ascending alphabetical order of the categories. When specifying release dates please use the format YYYY-MM-DD.

## 4.1. Criteria for Dead, Discontinued or Unmaintained Projects 

This is a bit difficult to say and it is handled on a case by case basis, but any of the following should be sufficient criteria

- No python 3 support
- Project website is no longer online
